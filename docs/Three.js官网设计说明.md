# Three.js 官网设计说明

## 1. 作品定位

官网命名为 InsightMesh AI，用于展示一家虚拟的智能检索科技公司。页面不是普通静态落地页，而是将公司主营业务“企业知识库混合检索 Agent”直接做成 3D 场景：文档节点、语义连接、Agent 决策核心和检索轨道共同构成可视化首页。

## 2. 技术选型

- Vite：轻量前端构建工具。
- Three.js：实现 3D 场景、材质、灯光、动画和交互。
- 原生 HTML/CSS：减少依赖，保证页面结构清晰。

## 3. 页面结构

| 区块 | 内容 |
| --- | --- |
| Hero | 公司名、产品定位、3D 知识网络、实时指标 |
| Company | 公司简介、服务对象、部署方式 |
| Solutions | 知识库检索中心、业务 Agent 助手、私有化 AI 中台 |
| Workflow | 理解问题、融合检索、工具执行、生成引用 |
| Cases | 研发团队、客服中心、人力行政典型客户场景 |
| Stack | Spring AI、PostgreSQL、pgvector、bge-m3、SSE、React、Three.js、LoRA |
| Contact | 公司联系入口 |

## 4. 3D 场景设计

### 4.1 中心核心

中心几何体代表 Agent 决策核心，使用青绿色材质和轻微自发光，表达系统正在持续思考和调度工具。

### 4.2 环形轨道

三层环形轨道代表 Agent Loop 中的任务规划、工具执行和结果验证。轨道以不同速度旋转，让用户直观看到执行循环。

### 4.3 文档节点

球体和薄片方块代表知识片段、制度文档和技术资料。节点围绕核心分布，并通过连接线与 Agent 核心相连。

### 4.4 语义连接

连接线代表检索召回路径。线段会跟随节点动画持续更新，形成“知识网络正在被检索”的视觉反馈。

## 5. 响应式设计

- 桌面端采用双列首屏，右侧显示运行指标。
- 移动端隐藏指标面板，保留品牌、导航、核心介绍和 3D 背景。
- 所有按钮和卡片设置稳定尺寸，避免动画或文字造成布局抖动。

## 6. 与 Agent 项目的关系

官网不是独立展示页，而是本次 Agent 作品的产品化表达：

- 首页 3D 网络对应混合检索知识图谱。
- Product 区块对应 Agent 设计说明中的核心能力。
- Workflow 区块对应 Think-Execute Loop 和 RAG 执行链路。
- Stack 区块对应后端、前端、检索和微调技术栈。
